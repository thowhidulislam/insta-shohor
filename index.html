How does Javascript work?

Javascript is a single threaded language because it can only execute one command at a time and follows a specific order. Javascript is a scripting language so it dosen't need compiler to translate its code. Javascript code runs directly in a web browser. Browser exucutes javascript in two phases. In first phase javascript scans all the code and allocates memory for all the variables and functions. While allocating the memory for variables it will store undefined and for functions it will keep the entire function code. In second phase javascript starts going through line by line.

What does Javascript event loop do?

Event loop does one simple job. It monitors the call stack and callback queue. Event loop maintains the communication with the call stack and call back queue. It checks whether the call stack is empty or not. When call stack is empty it takes the first event from the queue and pushes it to the call stack. Call back queue and call stack may be empty for some some time but event loop will never stop checking.

What is the differences between local and session storage?

There are not many differences exist between local and session storage. They almost do the same thing. The one difference they have is data storage duration. Local storage data won't expire once it is used in the browser. But session storage data expires or clears when that particular session ends. The data is stored for the duration of the session. The storage capacity of local storage is 5MB/10MB, while for the session is 5MB. Local storage data can be removed via javascript or manually.
